Live Trip Tracker New

Uber-style live location tracking for sales teams, entirely on platform. A rep starts their day and the browser's geolocation API streams GPS positions into Salesforce — buffered into 8-second batches, stored as trip points, and re-published as Trip_Ping__e platform events. LWC · Component Library · Refactor
Reusable LWC Component Library New

Three production components rebuilt into a fully self-contained, org-agnostic library — no server-side code, no managed packages, nothing to install first. errorHandlerUtility normalizes any wire, fetch, or platform error shape into a readable toast in one call — with a pluggable onLog hook so it works with whatever logging you already use, a withErrorHandling wrapper that removes try/catch boilerplate, and a retryable helper with exponential backoff for transient errors. recordHighlightPanel renders any record page's header from a single JSON config — fields, relationship fields via dot notation, and AND/OR visibility rules — plus an opt-in inline quick-edit mode, a collapse/expand toggle, a built-in Copy Link action, and currency/percent/date formatting for relationship fields. customUrlDatatable adds three column types lightning-datatable doesn't ship out of the box — a hyperlink cell, a colored status badge, and a percent progress bar — all driven purely by row data. Every component runs on base Lightning modules alone and deploys to any org in one line.

How to use
errorHandlerUtility
import { handleError } from 'c/errorHandlerUtility';

try { await save(); }
catch (e) { await handleError(this, e); }

One call per catch block — it parses the error, shows a readable toast, and logs it. Pass onLog to route into Nebula Logger or your own Log__c writer.

recordHighlightPanel
{ "iconName": "standard:opportunity",
  "fields": [
    { "fieldApiName": "StageName" },
    { "fieldApiName": "Account.Industry",
      "label": "Industry" } ],
  "actions": [
    { "apiName": "CopyLink", "label": "Copy Link",
      "showWhen": "true" } ] }

Drop it on any record page in App Builder and paste a JSON config — no code, no server-side setup. Visibility rules like StageName != 'Closed Lost' supported per field/action.

customUrlDatatable
columns = [{ label: 'Doc', type: 'customUrl',
  typeAttributes: {
    url: { fieldName: 'docUrl' },
    labelVal: { fieldName: 'docLabel' } } }];

Use <c-custom-url-datatable> instead of lightning-datatable and three extra column types unlock: customUrl, badge, and percentBar.

Deploy everything in one line: sf project deploy start --source-dir force-app — base Lightning modules only, works in any org.
